## 极简动漫序列推荐系统 (Mini-Anime-Rec) 项目计划书

### 1. 项目概述

本项目旨在开发一个轻量级的动漫推荐 Web 应用。核心目标不是处理海量数据，而是通过敏捷开发流程，跑通从前端交互、后端服务、深度学习模型推理到自动化测试与容器化部署的**完整软件工程生命周期**。

- **数据来源：** Kaggle Anime Recommendations Database (截取 Top 1000 动漫与活跃用户)。
- **核心功能：** 用户登录、浏览动漫列表、模拟点击打分、系统实时返回 Top-K 个性化推荐结果。

### 2. 技术栈与架构设计

为了实现高效率开发与轻量级部署，项目采用以下技术选型：

- **前端 (Frontend)：** Vue 3 或 React (轻量级脚手架构建)。
- **后端 (Backend)：** FastAPI (Python 生态，原生支持异步，极度契合机器学习模型调用)。
- **数据库 (Database)：** SQLite (单文件存储，零配置，随代码打包)。
- **模型工程 (Model Eng)：** PyTorch 离线训练 $\rightarrow$ 导出 ONNX 格式 $\rightarrow$ FastAPI 在线加载推理。
- **部署 (DevOps)：** Docker & Docker Compose (一键环境隔离与启动)。

### 3. 核心算法设计 (基于序列的推荐)

突破传统的矩阵分解，本项目将用户对动漫的交互历史视为标准的时间序列数据 $S = \{i_1, i_2, ..., i_t\}$。

在序列特征提取上，引入 **Geometry-Aware Temperature-Evidential (GATE) Network** 的架构思想。利用其在时间序列预测上的优势，将网络中的几何空间感知和温度证据机制平移到推荐场景中。这不仅能有效捕捉用户兴趣的时序波动，还能对推荐结果的不确定性进行精准建模和融合打分，极大提升算法模块的学术深度与工程创新性。

### 4. UI 与视觉交互规范

前端界面摒弃繁冗的装饰，严格遵循**“少即是多” (Less is More)** 的设计哲学。

- **视觉风格：** 采用纯粹的**扁平化设计 (Flat Design)**。
- **元素要求：** 舍弃多余的阴影和渐变，使用**极简线条**划分内容区块。
- **色彩管理：** 保持**克制色彩**，以大面积留白配合单一主色调（如低饱和度的蓝色或深灰），将视觉焦点完全让渡给 API 返回的动漫封面图片和排版文本。

### 5. 敏捷开发里程碑 (Sprint 规划)

项目开发周期划分为 5 个 Sprint（冲刺），每个 Sprint 包含明确的产出物。

| **Sprint 阶段**          | **核心目标**           | **关键任务与产出物 (Deliverables)**                          |
| ------------------------ | ---------------------- | ------------------------------------------------------------ |
| **Sprint 1: 需求与基建** | 完成架构设计与数据准备 | 1. 产出 API 接口文档 2. 编写 Python 脚本将 CSV 截断并转入 SQLite |
| **Sprint 2: 模型微服务** | 跑通推荐算法的核心逻辑 | 1. 离线训练 GATE 等序列模型并导出 ONNX 2. FastAPI 加载模型，编写接口接收张量并返回结果 |
| **Sprint 3: 业务后端**   | 完善数据库交互与 API   | 1. 编写用户历史记录查询 SQL 逻辑 2. 融合模型输出，返回完整的 JSON 数据流 |
| **Sprint 4: 前端联调**   | 完成用户界面与数据渲染 | 1. 按照扁平化规范搭建前端页面 2. 跨域调用后端 API，实现点击与推荐列表动态刷新 |
| **Sprint 5: QA与部署**   | 质量保证与一键交付     | 1. 编写 Pytest 单元测试与接口可用性测试 2. 编写 Dockerfile，实现全栈一键 `docker-compose up` |

### 6. 团队角色与质量保证 (QA)

在项目管理中，明确的分工是推进进度的关键：

- **项目经理 / 算法工程师：** 负责进度把控、撰写本计划书、训练 GATE 网络模型并完成 ONNX 转换。
- **后端工程师：** 负责搭建 FastAPI 框架，编写 SQL 语句，集成模型。
- **前端工程师：** 负责界面开发、路由跳转与接口对接。
- **测试与部署工程师：** 负责编写测试用例、撰写 Docker 脚本以及最终的演示环境搭建。
